## Overview

The Delay Block is used to introduce a delay in the execution of the flow. It takes any number of inputs and after a specified delay, passes the input values to the outputs without any modifications. This block is useful when you need to control the timing of your flow's execution.

## Example 1: Delaying a Message

1. Create a Text block and set its value to "Hello, World!".
2. Create a Delay block and set its "Delay" setting to 5000 (5 seconds).
3. Connect the output of the Text block to the first input of the Delay block.
4. Run the flow. Note that the output of the Delay block is not displayed until after 5 seconds.

## Error Handling

The Delay block cannot error under normal circumstances. If the delay time is negative, it will be treated as zero.

## FAQ

**Q: What happens if I set the delay time to zero?**

A: The Delay block will pass the input to the output immediately, effectively acting as a pass-through block.

**Q: Can I use the Delay block to throttle API calls?**

A: Yes, you can use the Delay block to throttle API calls. For example, if you have a flow that makes an API call every time it is run, you can use the Delay block to ensure that the API call is not made more than once every 5 seconds.
