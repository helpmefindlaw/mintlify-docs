## Overview

The HTTP Call Block allows you to make an HTTP call to a specified URL with a given method, headers, and body. This block is particularly useful when you need to interact with external APIs or services.

The HTTP Call Block uses the Fetch API to make HTTP requests. It supports all HTTP methods (GET, POST, PUT, DELETE, etc.) and allows you to specify custom headers and a request body.

## Example 1: Make a GET request to an API

1. Create an HTTP Call Block and set the `Method` to `GET` and the `URL` to `https://jsonplaceholder.typicode.com/todos/1`.
2. Run the flow. You should see all of the response data, such as the headers, response code, and body, in the output panel.

![HTTP Call Block Example 1](./assets/http-call-block-example-01.png)

## Example 2: Make a POST request to an API

1. Create an HTTP Call Block and set the `Method` to `POST` and the `URL` to `https://jsonplaceholder.typicode.com/posts`. Enable the "Use Input" toggle on the "Body" setting to enable the input port for Body.
2. Create an [Object Block](./object.mdx) and set the object to:

```json
{
  "title": "foo",
  "body": "bar",
  "userId": 1
}
```

3. Create an [Extract Object Path](./extract-object-path.mdx) Block and connect the `JSON` output of the HTTP Call Block to the `Object` input of the Extract Object Path Block. Set the `Path` to `$.id`.
4. Run the flow. You should see the created ID of the post in the Extract Object Path Block's output panel.

![HTTP Call Block Example 2](./assets/http-call-block-example-02.png)

## Error Handling

The HTTP Call Block will error if the HTTP request fails for any reason, such as a network error or if the server returns an error status code. If the `Error on non-200 status code` setting is enabled, the block will also error if the status code of the HTTP response is not 200.

## FAQ

**Q: Can I use the HTTP Call Block to make requests to any API?**

A: Yes, you can use the HTTP Call Block to make requests to any API that supports the HTTP methods GET, POST, PUT, or DELETE.

**Q: Can I use the HTTP Call Block to send JSON in the request body?**

A: Yes, you can use a [Text Block](./text.mdx) to create a JSON string and connect it to the `Body` input of the HTTP Call Block.

**Q: Can I use the HTTP Call Block to handle API authentication?**

A: Yes, you can include authentication headers in the `Headers` input of the HTTP Call Block. However, for security reasons, you should not hardcode sensitive information like API keys in your flows. Instead, consider using a [Context Block](./context.mdx) to securely pass in sensitive information from the host application.
