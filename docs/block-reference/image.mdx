## Overview

The Image Block is used to define a static image for use with other blocks. It can convert a binary type into an image type. The block requires a media type (PNG, JPEG, or GIF) and a data input.

The data input can be provided in two ways:

- Directly in the block's editor settings, using the image browser.
- Dynamically, by connecting another block to the Image Block's `Data` input port.

The media type can also be provided in two ways:

- Directly in the block's editor settings, using the dropdown menu.
- Dynamically, by connecting another block to the Image Block's `Media Type` input port.

## Inputs

| Title      | Data Type | Description                                      | Default Value                                               | Notes                                                                                                                                             |
| ---------- | --------- | ------------------------------------------------ | ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| Data       | `string`  | The data to be converted into an image.          | (required if if the input toggle for Data is enabled)       | The input will be coerced into a string if it is not a string.                                                                                    |
| Media Type | `string`  | The media type of the image (PNG, JPEG, or GIF). | (required if if the input toggle for Media Type is enabled) | The input will be coerced into a string if it is not a string. The value must be one of the following: `image/png`, `image/jpeg`, or `image/gif`. |

## Example 1: Create a static image

1. Create an Image Block.
2. In the block's editor settings, click on the `Pick Image` button. This will open the image browser.
3. Select an image from the image browser. The selected image will be displayed in the `Image` field.
4. Run the flow. The `Image` output of the Image Block should contain the selected image.

![Image Block Example 1](./assets/image-block-example-01.png)

## Error Handling

The Image Block will error if the `Data` input or the `Media Type` input is not provided. It will also error if the `Data` input cannot be converted into an image.

## FAQ

**Q: Can I use the Image Block to display an image in the Lawme UI?**

A: Yes, the Image Block can be used to display an image in the Lawme UI. The image will be displayed in the block's output port when the flow is run.

**Q: Can I use the Image Block to convert a binary type into an image?**

A: Yes, you can use the Image Block to convert a binary type into an image. You will need to provide the binary data to the `Data` input of the Image Block and set the `Media Type` to the appropriate type for the image.

**Q: What image formats are supported by the Image Block?**

A: The Image Block supports PNG, JPEG, and GIF formats.
