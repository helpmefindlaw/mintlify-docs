## Overview

The Multi-Modal Draft Block leverages AI to generate various types of documents based on user prompts, context, and images. It offers a set of example templates for common document types and an optimized custom template feature for more specific drafting needs. This block intelligently incorporates both textual and visual information to create comprehensive and context-aware documents.

## Inputs

<ParamField path="Context" type="object">
  An object containing key-value pairs for interpolation into the template. This provides additional information or context to be considered in the document drafting process.
</ParamField>

<ParamField path="Images" type="array" optional>
  An array of image objects to be analyzed and incorporated into the document. Each image object should contain a URL or base64-encoded image data.
</ParamField>

## Outputs

<ResponseField name="Draft" type="string">
  The generated document draft based on the input template, context, and images.
</ResponseField>

## Editor Settings

<ParamField path="Document Type" type="string" default="Custom">
  Select from predefined document types or choose "Custom" for a user-defined template.
</ParamField>

<ParamField path="Example Templates" type="string">
  A selection of example templates for common document types. Only visible when a predefined document type is selected.
</ParamField>

<ParamField path="Custom Template" type="string">
  An optimized template for custom document drafting. Only visible when "Custom" is selected as the document type.
</ParamField>

<ParamField path="AI Model" type="string" default="GPT-4">
  The AI model to use for document generation. Ensure the selected model supports multi-modal inputs.
</ParamField>

## Example Document Types

The Multi-Modal Draft Block provides example templates for the following document types:

1. Legal Briefs with Evidence Analysis
2. Product Descriptions with Visual Details
3. Research Reports with Data Visualization
4. Travel Guides with Landmark Recognition

## Example: Drafting a Product Description

1. Add a Multi-Modal Draft Block to your flow.
2. In the block's settings, select "Product Description" as the Document Type.
3. Choose an appropriate example template or modify it as needed.
4. Connect an Object block with product details to the "Context" input.
5. Connect an array of product images to the "Images" input.
6. Run the flow. The Draft Block will output a generated product description incorporating both textual and visual information.

## Custom Drafting

For more specific document needs:

1. Select "Custom" as the Document Type.
2. Use the optimized custom template provided, filling in the necessary details.
3. Add context and images as needed.
4. Run the flow to generate your custom document draft.

## Error Handling

The Multi-Modal Draft Block will return an error if:
- The input template is empty or too vague.
- The AI model fails to generate a coherent document.
- The generated content exceeds the model's maximum token limit.
- Provided images are in an unsupported format or cannot be processed.

<Warning>
  Always review and edit AI-generated content for accuracy, coherence, and appropriateness before use in professional or legal contexts.
</Warning>

## FAQ

<AccordionGroup>
  <Accordion title="How does the block handle image inputs?">
    The block uses advanced AI capabilities to analyze and interpret the content of input images. It then incorporates relevant visual information into the generated document, enhancing the overall quality and context of the output.
  </Accordion>

  <Accordion title="Can I use the Multi-Modal Draft Block for languages other than English?">
    Yes, the Multi-Modal Draft Block can generate content in multiple languages. Specify the desired language in your template for best results. Note that image analysis capabilities may vary by language.
  </Accordion>

  <Accordion title="How long can the generated documents be?">
    The length of generated documents is limited by the token limit of the chosen AI model. For longer documents, consider breaking the task into sections or using multiple Multi-Modal Draft Blocks.
  </Accordion>
</AccordionGroup>
