## Overview

The "Context" in Lawme is a set of shared inputs that are accessible in any flow, no matter if they are the entry-point flow, or a subflow called by another flow. The Context block is used to access these inputs.

The Context Block allows you to access these values in your flow. It is useful for "global values" that every flow can access, without having to pipe the values in as flow inputs. Use the Context Block and `contextValues` to pass in "global context" to your projects, such as the current date.

Context can currently only be set when Lawme is embedded in a host application. It is passed in using the `contextValues` parameter to `runFlowInFile` or `createProcessor`:

```ts

const contextValues = {
  stringContext: 'str',
  numberContext: 1,
  booleanContext: true,
};

const processor = Lawme.createProcessor({
  ...etc,
  contextValues,
});
```

## Example 1: Access a string context value

In your parent application, pass a string value into `contextValues`:

```ts

const contextValues = {
  stringContext: 'str',
};

const processor = Lawme.createProcessor({
  ...etc,
  contextValues,
});
```

In your flow, add a Context block. Set the `ID` to `stringContext`, and the `Data Type` to `String`. Leave the `Default Value` empty.

Run your flow while the [Remote Debugger](../user-guide/workflow/remote-debugging.md) is attached. The output of the Context block should be `str`.

## Error Handling

The context block cannot error in normal circumstances. If the context is not set via `contextValues`, then the default value will be used.

## FAQ

**Q: Can I use the Context block to pass in the current date and time?**

A: Yes, you can pass in the current date as a context value:

```ts
const processor = Lawme.createProcessor({
  ...etc,
  contextValues: {
    currentDate: new Date(),
  },
});
```

Then, in your flow, add a Context block. Set the `ID` to `currentDate`, and the `Data Type` to `Date`. Leave the `Default Value` empty.

**Q: Can I use the Context block to pass in a function?**

A: No, you cannot pass in a function as a context value. You can only pass in [Data Values](../user-guide/workflow/data-types.md)

**Q: Can I use the Context block to pass in a custom data type?**

A: No, you cannot pass in a custom data type as a context value. You can only pass in [Data Values](../user-guide/workflow/data-types.md)
