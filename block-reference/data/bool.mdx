## Overview

The Bool Block is used to output a boolean constant or convert an input value into a boolean. It's particularly useful when you need to create a boolean value for use in other blocks, such as the [If Block](/block-reference/data/if) or [If/Else Block](/block-reference/data/if-else).

## Inputs

<ParamField path="Input" type="any">
  The value to be converted to a boolean. This input is only available if the "Use Input" setting is enabled.
</ParamField>

## Outputs

<ResponseField name="Value" type="boolean">
  The boolean value. If "Use Input" is enabled, this will be the input value converted to a boolean.
</ResponseField>

## Editor Settings

<ParamField path="Value" type="boolean" default={false}>
  The constant boolean value to output. This setting is only available if "Use Input" is disabled.
</ParamField>

<ParamField path="Use Input" type="boolean" default={false}>
  If enabled, the block will convert an input value to a boolean. If disabled, the block will output a constant boolean value.
</ParamField>

## Conversion Rules

The Bool Block follows JavaScript's rules for truthy and falsy values, with two exceptions:

- Empty arrays are considered falsey
- The string "false" is considered falsey

If you need to treat empty arrays or the string "false" as truthy, you can use the [Code Block](/block-reference/data/code).

## Example: Using a Constant Boolean Value

1. Add a Bool Block to your flow.
2. Set the `Value` setting to `true`.
3. Run the flow. The `Value` output of the Bool Block will be `true`.

## Example: Converting an Input to a Boolean

1. Add a Bool Block to your flow.
2. Enable the `Use Input` setting.
3. Add a [Text Block](/block-reference/data/text) to your flow and set the text to `"Hello"`.
4. Connect the output of the Text Block to the `Input` of the Bool Block.
5. Run the flow. The `Value` output of the Bool Block will be `true`, because the string `"Hello"` is a truthy value in JavaScript.

## Error Handling

The Bool Block cannot error under normal circumstances. If the `Use Input` setting is enabled and no input is connected, the block will output `false`.

## FAQ

<AccordionGroup>
  <Accordion title="What values are considered truthy and falsy in JavaScript?">
    The following values are considered falsy:
    - `false`
    - `0`
    - `''` (empty string)
    - `null`
    - `undefined`
    - `NaN`
    - In Lawme, empty arrays are also considered falsy.
    - In Lawme, the string "false" is also considered falsy.

    All other values are considered truthy.
  </Accordion>

  <Accordion title="Can I use the Bool Block to check if an array is empty?">
    Yes, if you connect an array to the `Input` of a Bool Block, the `Value` output will be `false` if the array is empty. If the array is not empty, the `Value` output will be `true`.
  </Accordion>

  <Accordion title="Can I use the Bool Block to check if a string is empty?">
    Yes, you can connect a [Text Block](/block-reference/data/text) to the `Input` of a Bool Block. If the string is empty, the `Value` output of the Bool Block will be `false`. If the string is not empty, the `Value` output will be `true`.
  </Accordion>

  <Accordion title="Can I use the Bool Block to check if a number is zero?">
    Yes, you can connect a [Number Block](/block-reference/data/number) to the `Input` of a Bool Block. If the number is zero, the `Value` output of the Bool Block will be `false`. If the number is not zero, the `Value` output will be `true`.
  </Accordion>
</AccordionGroup>
