## Overview

The Filter Block is used to filter an array based on a corresponding array of boolean values. The block takes in an array of any data type and an array of boolean values of the same length. It then filters the array, only including the elements where the corresponding boolean value is `true`.

This block is particularly useful when you want to filter an array based on some condition. For example, you could use a batched block (see [Batching](../user-guide/workflow/batching.md)) [Compare Block](./compare.mdx) to compare each element in an array to a value, and then use the Filter Block to filter the array based on the results of the comparison.

## Inputs

| Title   | Data Type   | Description                                                                         | Default Value | Notes                                                                                                                                             |
| ------- | ----------- | ----------------------------------------------------------------------------------- | ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| Array   | `any[]`     | The array to be filtered.                                                           | (required)    | The input will be coerced into an array if it is not an array.                                                                                    |
| Include | `boolean[]` | An array of boolean values indicating whether to include each element in the array. | (required)    | The input will be coerced into a boolean array if it is not a boolean array. The length of this array should match the length of the Array input. |

</TabItem>

<TabItem value="outputs">

## Outputs

| Title    | Data Type | Description                                                                                                             | Notes                                                                                                                                           |
| -------- | --------- | ----------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| Filtered | `any[]`   | The filtered array, containing only the elements where the corresponding boolean value in the Include input was `true`. | The output will be an array of the same data type as the Array input. If the Array input is not an array, the output will be an array of `any`. |

</TabItem>

## Example 1: Filter an array of numbers

1. Create an [Array Block](./array.mdx) and set the values to `[1, 2, 3, 4, 5]`.
2. Create another Array Block and set the values to `[true, false, true, false, true]`.
3. Create a Filter Block and connect the first Array Block to the `Array` input and the second Array Block to the `Include` input.
4. Run the flow. The `Filtered` output of the Filter Block should be `[1, 3, 5]`.

![Filter Block Example 1](./assets/filter-block-example-01.png)

## Example 2: Filter an array based on a condition

1. Create an [Array Block](./array.mdx) and set the values to `[1, 2, 3, 4, 5]`.
2. Create a [Compare Block](./compare.mdx) and set the comparison to `>`. Enable batching on the Compare block.
3. Create a [Number Block](./number.mdx) and set the value to `3`.
4. Connect the Array Block to the `A` input of the Compare Block, and connect the Number Block to the `B` input of the Compare Block.
5. Create a Filter Block and connect the Compare Block to the `Array` input and the `Result` output of the Compare Block to the `Include` input.
6. Run the flow. The `Filtered` output of the Filter Block should be `[4, 5]`.

![Filter Block Example 2](./assets/filter-block-example-02.png)

## Error Handling

The Filter Block will error if the `Array` or `Include` inputs are not provided, or if the lengths of the arrays do not match.

## FAQ

**Q: What happens if the lengths of the Array and Include inputs do not match?**

A: The Filter Block will error. The lengths of the Array and Include inputs should always match.

**Q: Can I use the Filter Block to filter an array of objects?**

A: Yes, you can use the Filter Block to filter an array of any data type, including objects. Just make sure to provide an array of boolean values indicating whether to include each object in the filtered array.
